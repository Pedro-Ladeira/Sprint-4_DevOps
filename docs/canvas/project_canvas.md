# Project Canvas

## Nome do projeto
Mottu API - Sistema de Gerenciamento de Motos

## Problema
Gerenciar a frota de motos, sensores e pátios de forma eficiente, com monitoramento de sensores e histórico de posições.

## Stakeholders
- Mottu (cliente)
- Usuários internos (operadores/admin)
- Equipe acadêmica

## Proposta de valor
Fornecer uma aplicação web para controlar, auditar e consultar o parque de motos, com dashboards e alertas de sensores.

## MVP
- CRUD de motos, modelos, pátios e sensores
- Dashboard básico
- Autenticação de usuários

## Métricas de sucesso
- Endpoints funcionando e testados
- Interface responsiva com boas práticas de UX
- Demonstração do fluxo principal em vídeo

## Recursos
- Java 17, Spring Boot, H2/Oracle, Docker

## Riscos
- Integração com Oracle em ambiente de produção
- Tempo para rodar testes/CI

- Completar documentação e diagramas
- Deploy público (Railway / Render / Heroku via container)
- Gravar vídeo com todos os membros
