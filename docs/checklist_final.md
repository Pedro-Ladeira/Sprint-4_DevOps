# Checklist Final da Entrega

Use este checklist para validar a entrega antes de enviar (marque ✅ quando concluído).

## Entrega técnica (40 pts)
- [ ] Aplicação rodando online (link público)
- [ ] Demonstração dos principais fluxos (login, CRUD de motos, dashboard)
- [ ] Uso de features da disciplina (Spring Data JPA, Validation, Security)
- [ ] Interface com boa UI/UX (responsiva)

## Narrativa (20 pts)
- [ ] README e docs com justificativas de design
- [ ] Decisões tecnológicas documentadas
- [ ] Evidência de originalidade

## Integração Multidisciplinar (20 pts)
- [ ] Canvas do projeto anexado
- [ ] Scripts SQL / migrações (Flyway) incluídos
- [ ] Diagramas (ER, Arquitetura, UML)

## Apresentação em vídeo (10 pts)
- [ ] Vídeo de apresentação (<= 15 min) com todos os membros
- [ ] Roteiro e tempo por membro documentado

## Organização e documentação (10 pts)
- [ ] README completo com quickstart
- [ ] Docs organizados em `/docs`
- [ ] CI configurado (build + test)

---

Observações finais:
- Verifique o `.gitignore` e remova artefatos binários do repositório.
- Não comite credenciais (use variáveis de ambiente ou secrets no CI).
- Entregas fora do prazo serão desconsideradas.
